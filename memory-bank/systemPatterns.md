# System Patterns: xdash

*   **Architecture**: 
    *   Core `xdash` library project.
    *   Documentation managed within the main project using VitePress.
    *   Core library remains modular, exporting utilities via `package.json#exports`.
*   **Design Patterns**: 
    *   Utility Functions: Collection of standalone functions.
    *   Type Guards: Runtime type checking.
    *   Functional Programming (FP): Functions reviewed/refactored for FP principles.
    *   Chainable Interface (`Chain` class).
    *   Inline Switch (`InlineSwitch` class).
*   **Code Structure**: 
    *   `src/`: Core TypeScript source code.
    *   `tests/`: Unit tests using Jest (`bun test`).
    *   `dist/`: Build output (not committed).
    *   `docs/`: VitePress source files (incl. generated API Markdown in `docs/api/`).
    *   `docs/.vitepress/`: VitePress configuration and components.
    *   `.github/workflows/`: CI/CD pipelines.
    *   `package.json`: Defines metadata, dependencies, scripts (test, build, docs).
    *   `build.ts`: Custom build script using Bun's build API.
    *   `typedoc.json`: Configuration for TypeDoc Markdown generation.
*   **API Design**: Supports namespace and named imports. Supports deep imports via `exports` map.
*   **Testing Pattern**: Unit testing with Jest (`bun test`). Benchmarking TBD.
*   **Documentation Pattern**: Static site generated by VitePress, using Markdown content generated by TypeDoc for API reference. Deployed to GitHub Pages.
*   **Release Pattern**: Automated releases to npm triggered by CI/CD.
*   **Deprecated Patterns**: Astro for docs, `release-it`.
