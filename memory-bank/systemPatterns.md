# System Patterns: xdash

*   **Architecture**: 
    *   Monorepo-like structure containing the core `xdash` library and a separate `astro/` project (likely for documentation).
    *   Core library is modular, exporting utilities through a main entry point (`dist/index.esm.js`) and specific module entry points (`dist/array.esm.js`, etc.) defined in `package.json#exports`.
*   **Design Patterns**: 
    *   Utility Functions: Provides a collection of standalone functions.
    *   Type Guards: Includes functions for runtime type checking (`isStr`, `isNum`, etc.).
    *   Likely functional programming influences (needs code review for confirmation, e.g., immutability).
*   **Code Structure**: 
    *   `src/`: Core TypeScript source code, organized by functionality (e.g., `array.ts`, `string.ts`).
    *   `tests/`: Unit tests mirroring the `src/` structure (`*.test.ts`).
    *   `dist/`: Build output (JavaScript modules and type definitions).
    *   `docs/`: Static documentation generated by TypeDoc.
    *   `astro/`: Separate Astro project, likely for the documentation website.
    *   `package.json`: Defines metadata, dependencies, scripts, and export map.
    *   `build.ts`: Custom build script using Bun's build API.
*   **API Design**: 
    *   Supports both namespace import (`import * as x from 'xdash'`) and named imports (`import { isStr } from 'xdash'`).
    *   Supports deep imports for specific modules (`import { map } from 'xdash/array'`) enabled by the `exports` field in `package.json`.
